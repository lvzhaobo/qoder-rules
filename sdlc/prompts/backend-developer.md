# 后端开发提示词 (Backend Developer)

## 角色定位
**后端开发** - 负责企业级 API 设计和后端业务逻辑实现

---

## 场景: 企业级 API 设计 (Enterprise API)

### 目标
遵循企业标准设计接口

### 提示词

```
你是一位 [公司名称] 的后端工程师。请设计"用户抢购"接口 `POST /seckill/order`。
约束条件：
1. **框架**：[Spring Boot + MyBatis Plus]。
2. **鉴权**：集成内部网关鉴权 `@RequirePermission("seckill.buy")`。
3. **异常处理**：使用公司通用的 `BizException` 和 `Result<T>` 包装器。
4. **实现**：伪代码需包含 Redis 分布式锁（使用 Redisson）防止超卖。

请提供 Controller 和 Service 层的核心逻辑。
```

### 预计输出
- 符合公司规范的伪代码
- 集成内部组件的示例
- 标准的 API 接口定义

### 效果
✅ 生成的代码直接可用，无需二次修改风格，符合安全合规要求。

---

## 使用建议

💡 **适用场景：**
- 新接口设计
- 复杂业务逻辑实现
- 高并发场景处理
- 企业规范落地

**注意事项：**
- 接口设计应遵循 RESTful 规范
- 鉴权、限流、熔断等中间件要提前配置
- 分布式锁要注意超时和释放机制
- 异常处理要统一，避免直接抛出底层异常
- 日志记录要包含关键业务参数，方便问题排查

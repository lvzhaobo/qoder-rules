# 产品经理提示词 (Product Manager)

## 角色定位
**产品经理** - 负责需求拆解、用户故事编写和数据埋点设计

---

## 场景: 复杂需求拆解 & 用户故事 (User Story)

### 目标
将模糊的业务诉求转化为可开发的标准文档

### 提示词

```
你是一位拥有 10 年经验的高级产品经理。请将以下业务需求转化为符合 INVEST 原则的敏捷用户故事。
业务需求："[我们需要一个积分商城，用户可以用积分换礼物]。"

请输出：
1. **用户故事** (As a... I want to... So that...)
2. **详细验收标准 (AC)**：请使用 Gherkin 语法 (Given/When/Then)，覆盖正常兑换、积分不足、库存不足、高并发扣减等场景。
3. **数据埋点**：基于 [神策/Google Analytics] 格式定义埋点事件。
4. **边缘案例**：列出至少 3 个容易被忽略的异常流程。
```

### 预计输出
- 标准的敏捷用户故事卡片
- Gherkin 格式的可测试验收标准
- 清晰的数据埋点定义

### 效果
✅ 开发人员能直接理解业务边界，测试人员可直接转化为测试用例。

---

## 使用建议

💡 **适用场景：**
- 需求评审前的文档准备
- 用户故事卡片编写
- 数据埋点方案设计
- 边缘案例梳理

**注意事项：**
- 用户故事应符合 INVEST 原则（Independent, Negotiable, Valuable, Estimable, Small, Testable）
- 验收标准要清晰可测试，避免模糊表述
- 埋点设计应在开发前完成，避免后期补埋
- 边缘案例往往是测试遗漏的重点

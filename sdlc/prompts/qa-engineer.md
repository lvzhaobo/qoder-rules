# 测试工程师提示词 (QA Engineer)

## 角色定位
**测试工程师** - 负责自动化测试用例生成和边界测试

---

## 场景: 自动化用例生成 & 边界测试

### 目标
生成健壮的集成测试脚本

### 提示词

```
针对"用户注册"接口，请设计自动化测试用例。
要求：
1. **工具**：使用 [Pytest + Requests / Postman]。
2. **覆盖范围**：Happy Path、参数校验、SQL 注入 Payload 尝试。
3. **环境适配**：从环境变量读取 `BASE_URL`，适应 Dev/Test/Prod 环境。
4. **数据驱动**：使用数据驱动模式加载 `test_data.csv`。
```

### 预计输出
- 可配置的自动化测试脚本
- 环境无关的测试代码
- 基础的安全测试 Payload

### 效果
✅ 自动化回归测试，确保新功能上线不破坏已有功能。

---

## 使用建议

💡 **适用场景：**
- 接口自动化测试
- 回归测试用例生成
- 边界场景覆盖
- 安全测试 Payload 准备

**注意事项：**
- 测试用例要覆盖正常流程、异常流程和边界条件
- 数据驱动测试可以提高测试覆盖率和维护性
- 环境配置要与实际环境分离，避免硬编码
- 安全测试要适度，不要在生产环境执行破坏性测试
- 测试数据要有代表性，覆盖典型场景

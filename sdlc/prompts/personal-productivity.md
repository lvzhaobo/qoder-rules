# 个人效能提示词 (Personal Productivity)

## 角色定位
**全员通用** - 适用于所有研发人员的个人效能提升场景

---

## 场景 1: 遗留代码理解 (Legacy Code)

### 目标
快速接手看不懂的"祖传代码"

### 提示词

```
我需要接手维护这段 [Java/Python] 遗留代码。请作为资深架构师：
1. **逻辑可视化**：用 Mermaid 流程图画出其核心执行路径。
2. **业务翻译**：用通俗的语言解释这段代码在做什么业务（注意：它似乎处理了 [订单状态流转]）。
3. **风险预警**：指出代码中硬编码的"魔术数字"和潜在的 NullPointerException 风险。
4. **重构建议**：如果不破坏现有逻辑，如何用 [Java 8 Stream] 简化它？

代码片段：
`[粘贴代码]`
```

### 预计输出
- 可视化的 Mermaid 流程图
- 通俗易懂的业务逻辑说明
- 防御性编程的改进建议

### 效果
✅ 降低新人上手老旧系统的门槛，减少因理解偏差导致的 Bug。

---

## 场景 2: 微任务自动化 (Toolkit)

### 目标
解决正则、Cron、JSON转换等琐碎痛点

### 提示词

```
我需要处理一个文本匹配任务。
需求：匹配 [符合公司工号格式（W+6位数字）] 的字符串。
请输出：
1. **正则表达式**：兼容 [JavaScript] 环境。
2. **测试用例**：提供 3 个匹配成功和 3 个匹配失败的样本。
3. **解释**：逐段解释正则含义。
4. **代码片段**：生成一个 TypeScript 函数 `isValidEmployeeId(id: string)`。
```

### 预计输出
- 精准的正则表达式
- 开箱即用的验证函数
- 完备的测试数据

### 效果
✅ 将开发者从记忆琐碎语法中解放出来，专注于核心逻辑。

---

## 场景 3: 报错排查 (Debugging)

### 目标
根据日志快速定位根因

### 提示词

```
我的程序报错了，堆栈信息如下。
环境：[Spring Boot 2.7, JDK 11, MySQL 8]。
最近我修改了：[数据库连接池配置]。

堆栈信息：
`[粘贴 StackTrace]`

请分析：
1. **直接原因**：是什么具体的异常？
2. **关联推测**：这是否和我修改的连接池配置有关？
3. **解决方案**：给出 2 种可能的修复方案（临时规避 vs 根治）。
```

### 预计输出
- 异常原因的深度解析
- 排查思路的引导
- 可复制的修复代码

### 效果
✅ 缩短 MTTR（平均修复时间），提升开发者的独立解决问题能力。

---

## 使用建议

💡 **企业级使用指南：** 为了获得最佳效果，请在使用 Prompt 前注入企业上下文。
建议将通用规则配置在 IDE 的 System Prompt 中：
```
"你是一位就职于 [公司名称] 的高级工程师。我们的技术栈是 [Spring Boot 2.7 / React 18]，内部封装库为 [MyCorp-Common-Lib]。请遵循 [阿里巴巴Java开发手册] 规范。回答请保持简洁专业。"
```

**注意事项：**
- 红色标注的内容 `[...]` 需要根据实际情况替换
- 建议保存常用的占位符配置，提高复用效率

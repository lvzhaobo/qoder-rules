# 项目经理提示词 (Project Manager)

## 角色定位
**项目经理** - 负责任务拆解、进度管理和风险控制

---

## 场景 1: WBS 任务拆解 (Task Breakdown)

### 目标
将需求拆解为可执行的 Jira 任务

### 提示词

```
我们即将启动 [会员积分商城 v1.0] 项目。核心功能包括：积分赚取、商品兑换、积分过期。
请作为资深 PM：
1. **任务拆解**：生成一份 WBS (工作分解结构)，拆分到 0.5~2 人日的粒度。
2. **估时**：为每个任务预估开发工时 (Man-Hours)。
3. **依赖**：标出任务间的前置依赖关系。
4. **格式**：输出为 Markdown 表格，包含列：[模块, 子任务, 描述, 预估工时, 依赖任务, 推荐人员角色]。
```

### 预计输出
- 结构化的 WBS 任务表
- 工时预估参考
- 任务依赖关系图

### 效果
✅ 实现精细化管理，避免漏项，辅助排期。

---

## 场景 2: 进度与风险分析 (Progress & Risk)

### 目标
基于周报识别项目风险

### 提示词

```
这是团队本周的日报汇总（见附录）。项目截止日期是下周五。
请分析：
1. **进度偏差**：哪些模块滞后了？滞后原因是什么（技术难点/需求变更）？
2. **风险预警**：是否存在关键路径阻塞？（如：前端在等后端接口）。
3. **调整建议**：是否需要协调资源或砍掉低优先级功能？请给出 3 个可行的纠偏方案。
4. **周报摘要**：生成一份给 CTO 看的精简版项目周报。

附录：`[粘贴团队日报]`
```

### 预计输出
- 项目健康度诊断
- 风险应对策略
- 高层汇报摘要

### 效果
✅ 主动识别风险，从"事后救火"转变为"事前管理"。

---

## 使用建议

💡 **适用场景：**
- 项目启动阶段的任务规划
- 每周/每日的进度跟踪
- 项目风险识别与应对
- 向上汇报的材料准备

**注意事项：**
- WBS 拆解要足够细，但不要过度拆解
- 估时应基于团队历史数据，并留有缓冲
- 关键路径上的任务要重点关注
- 风险识别要前置，不要等问题发生后才处理

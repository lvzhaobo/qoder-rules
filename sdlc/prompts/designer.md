# UI/UX 设计师提示词 (Designer)

## 角色定位
**设计师/产品** - 负责 UI 原型设计和快速验证

---

## 场景: UI 原型设计 (Prototyping)

### 目标
快速输出高保真原型代码

### 提示词

```
请设计一个"企业级数据监控仪表盘"的 UI 原型。
1. **工具**：HTML + Tailwind CSS (通过 CDN 引入)。
2. **布局**：左侧侧边栏导航，顶部 Header，右侧内容区。
3. **组件**：
   - 4 个关键指标卡片 (KPI Cards)。
   - 一个占满宽度的折线图区域 (用 Placeholder 图片或 CSS 模拟)。
   - 一个最近告警列表 (Table)。
4. **风格**：深色模式 (Dark Mode)，科技感配色。
```

### 预计输出
- 响应式的 HTML/CSS 原型
- 现代化的 UI 布局

### 效果
✅ 无需 Figma 即可快速验证 UI 布局想法，加速产品评审。

---

## 使用建议

💡 **适用场景：**
- 快速原型验证
- UI 布局探索
- 产品评审演示
- 前端开发参考

**注意事项：**
- 原型重在快速验证，不要过度追求细节
- 使用现代 CSS 框架可以快速搭建布局
- 深色模式要注意对比度和可读性
- 响应式设计要考虑移动端适配
